# Question Types

## Purpose

Define when each question type should be used, what it is good at, and when it should be avoided.

This prevents the engine from defaulting to the same multiple-choice visual pattern for everything.

## Multiple Choice

Best for:

- direct recall
- single-concept distinction
- quick adaptive rounds
- visual recognition when the image is actually necessary

Use when:

- the learner must choose one best answer
- distractors can be made plausible without being deceptive

Avoid when:

- the question becomes trivial because the diagram labels the answer
- a plain definition prompt would test the concept more cleanly

## Matching

Best for:

- term -> definition
- normal form -> what it removes
- integrity rule -> wording
- relational term -> common synonym
- chapter concept -> short description

Use when:

- breadth matters
- the learner is already performing well and needs faster concept coverage
- you want to reduce repetition from multiple-choice-only rounds
- the matching choices can be shuffled without changing the meaning of the question

Avoid when:

- one side contains ambiguous overlaps
- the items are too long and become visually noisy
- the set mixes three tightly related items with one unrelated outlier that makes elimination too easy
- the same right-side ordering would make the answer obvious by pattern rather than knowledge

Matching sets should usually be either:

- all from one coherent subject area, or
- intentionally broad but balanced across the whole set

## True/False

Best for:

- textbook traps
- common misconceptions
- quick recognition checks

Use when:

- the statement is crisp
- the explanation can cleanly show why it is true or false

Avoid when:

- the wording depends on tiny technical ambiguity
- the statement would be better tested as a fuller distinction question

## Scenario Multiple Choice

Best for:

- normalization problems
- key selection
- relationship implementation
- higher-difficulty application questions

Use when:

- the learner needs to apply a concept, not just define it
- the scenario is short and concrete

Avoid when:

- the setup becomes longer than the actual learning point

## Visual Recognition

This is not a separate schema type by itself. It is usually a multiple-choice or scenario question that happens to include a diagram.

Best for:

- crow's foot cardinality
- unary/binary/ternary recognition
- supertype/subtype
- PK/FK placement
- bridge entity recognition

Use when:

- the visual is the thing being tested

Avoid when:

- the image only decorates a question that should be plain text
- labels inside the image give away the answer
- the image must carry too much annotation to make sense

## Recommended Mix by Performance Level

### Remediation Mode

Use more:

- multiple choice
- scenario multiple choice
- a few visuals for high-value concepts

Use less:

- matching overload

### Mixed Mode

Use:

- multiple choice
- matching
- scenario prompts
- moderate visual variety

### High-Performance Mode

Use more:

- matching
- distinction questions
- definition questions
- under-covered topics
- some higher-difficulty scenarios

Use less:

- repeated basic visual drills on the same pattern

## Design Rule

Not every question needs a visual element.

The engine should deliberately choose the simplest type that tests the concept cleanly.

However, when the topic is inherently visual — charts, diagrams, patterns, topologies, flowcharts — a visual is not optional. If the question asks about something you would explain with a picture, include the picture. Questions about candlestick charts, Head and Shoulders patterns, network diagrams, ER models, etc. must have a diagram. Asking a learner to identify parts of a chart they cannot see defeats the purpose.
